# 27 Markets — Real Trading (MetaTrader 5‑backed) Design

Date: 2026-06-26
Status: Phase 1 build. Real application layer + execution seam; SIMULATION execution
active now, **MetaTrader 5 white‑label adapter** is the live target.

## Goal
A real‑world trading web application. Most of it is already built (auth/2FA, KYC,
funding + Stripe seam, accounts, live market data + candlesticks, CRM, dashboards).
This adds the **trading** layer (orders, positions, P&L) behind an **ExecutionProvider
seam**, so it runs on simulation today and routes **real orders to MetaTrader 5** once
the licensed white‑label server + gateway credentials are connected.

## Architecture — ExecutionProvider seam (mirrors the payment/Stripe seam)
```
TradingService ──> EXECUTION_PROVIDER (DI token)
   ├─ SimulationExecutionProvider   fills at the live MarketService price (DEMO accounts)
   └─ Mt5ExecutionProvider          routes to the MT5 gateway (LIVE) — gated by MT5_* env
```
- `ExecutionProvider.fill(symbol, side, qty)` → `{ price, simulated }` (market fill).
- Selected by `EXECUTION_PROVIDER` (`simulation` default | `mt5`). MT5 refuses without
  `MT5_GATEWAY_URL` + credentials (like `UnavailableLivePaymentProvider`).
- Going live: stand up a licensed **MT5 white‑label** server, set `EXECUTION_PROVIDER=mt5`
  + gateway creds, and the adapter places real trades / syncs positions. No app rewrite.

## Phase 1 scope (this build)
- **Market orders** (BUY/SELL) on **demo accounts**, filled at the real live price.
- **Positions**: open via order, live unrealized P&L (client‑side from the price stream),
  close at live price → realized P&L posted to the demo ledger (simulated).
- **Order history** with status.
- **MT5 adapter stub** + config (so the live path is wired, not implemented).

Deferred (Phase 2+): limit/stop orders, margin + stop‑out/risk engine, the actual MT5
gateway calls + account/position sync, leverage enforcement, partial closes.

## Data model (Prisma)
```prisma
enum OrderSide { BUY SELL }
enum OrderType { MARKET }           // LIMIT/STOP later
enum OrderStatus { FILLED REJECTED CANCELLED }
enum PositionStatus { OPEN CLOSED }

model Order {
  id        String  @id @default(cuid())
  userId    String
  accountId String                  // TradingAccount
  symbol    String
  side      OrderSide
  type      OrderType   @default(MARKET)
  quantity  Decimal     @db.Decimal(18, 8)
  price     Decimal     @db.Decimal(18, 8)   // fill price
  status    OrderStatus @default(FILLED)
  simulated Boolean     @default(true)
  createdAt DateTime    @default(now())
  @@index([userId]); @@index([accountId])
}

model Position {
  id           String   @id @default(cuid())
  userId       String
  accountId    String
  symbol       String
  side         OrderSide
  quantity     Decimal  @db.Decimal(18, 8)
  entryPrice   Decimal  @db.Decimal(18, 8)
  exitPrice    Decimal? @db.Decimal(18, 8)
  realizedPnl  Decimal? @db.Decimal(18, 2)
  status       PositionStatus @default(OPEN)
  openedAt     DateTime @default(now())
  closedAt     DateTime?
  @@index([userId]); @@index([accountId, status])
}
```
User gains back‑relations `orders` + `positions`.

## API (client, authenticated)
- `POST /trading/orders` `{ accountId, symbol, side, quantity }` → opens a position (market).
- `GET  /trading/positions?status=OPEN` → open positions.
- `POST /trading/positions/:id/close` → close at live price, realize P&L.
- `GET  /trading/orders` → order history.
Simulation path enforces **demo accounts only**; live path will defer to MT5.

## Frontend
- `tradingApi` (place/close/list).
- **Trade** page (`/portal/trade`): instrument + order ticket (side, quantity, market),
  open positions table with **live unrealized P&L** (from the quote stream) + close,
  and order history with status. Nav entry added.

## Security / safety
- All trading routes behind JWT. Simulated trades only on DEMO accounts; SIMULATION rail
  stays on — no real funds until MT5 + licensing are live (`ALLOW_LIVE_MODE` gate).
- Realized P&L moves only via balanced ledger entries.
